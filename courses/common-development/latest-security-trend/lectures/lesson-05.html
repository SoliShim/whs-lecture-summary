SECTION 01

Web 3.0의 의미

강의는 Web 3.0 시대라는 표현에서 시작한다. 설명의 흐름은 Web 2.0에서 Web 3.0으로의 변화다. Web 2.0의 동력은 스마트폰, 모바일 인터넷, 이커머스, SNS의 부상이었고, 새로운 비즈니스 모델과 상시 연결 상태의 고객을 만들었다.

Web 3.0은 탈중앙화 플랫폼, 사용자가 소유하는 커뮤니티 중심 프로젝트, 뛰어난 개발자 생태계를 바탕으로 등장한다. 본질적으로는 개인 사용자에게 데이터와 디지털 삶에 대한 더 많은 투명성, 접근성, 자율성, 통제력을 제공하려는 흐름이다.

Web 3.0은 개인이 데이터와 정보에 대한 통제권을 되찾고, 자신이 만든 콘텐츠를 수익화하며, 공통 관심사와 목표를 공유하는 사람들과 더 쉽게 조직화할 수 있는 분산적 온라인 경험을 지향한다. 블록체인을 기반으로 암호화폐, 탈중앙화 금융, DAO, 메타버스 등이 중추 기능을 이룬다.

SECTION 02

오픈 블록체인과 Web 3.0의 비전

이더리움이나 솔라나 같은 블록체인 프로토콜은 거대한 공유 컴퓨터처럼 동작한다. 누구나 스마트 계약을 블록체인에 배치하고 사용할 수 있으며, 블록체인에 저장되는 데이터는 네트워크 참여자의 합의로 정확성과 무결성이 유지된다.

오픈 블록체인에 저장된 정보는 투명하고, 변경하기 어렵고, 추적 가능하다. 그래서 Web 1.0의 비저장성 격차와 Web 2.0의 중앙집중화 문제를 해결할 수 있다고 설명된다. 더 분산화된 인터넷은 소규모 온라인 커뮤니티에도 영향력을 주고, 커뮤니케이션과 조직 구성을 효율적으로 만들 수 있다.

SECTION 03

블록체인의 네 가지 특징
특징설명
분산화트랜잭션 검증에 중앙 서버만 의존하지 않고, 합의 기법으로 분산 네트워크의 데이터 일관성을 유지한다.
불변성트랜잭션이 블록체인에 기록되면 삭제하거나 되돌리기 거의 어렵다. 유효하지 않은 트랜잭션은 노드가 받아들이지 않는다.
익명성사용자는 실제 신원 정보가 직접 포함되지 않는 유사 익명 주소로 블록체인과 상호작용한다.
추적성트랜잭션은 이전 미사용 트랜잭션을 참조하므로 흐름을 확인하고 추적할 수 있다.
SECTION 04

블록체인 기술 발전 흐름

2009년 이후 블록체인 기반 기술 발전은 다섯 가지로 요약된다.

기술설명
암호화폐비트코인, 피어코인처럼 보존과 유통을 위해 설계된 가상화폐다.
자산 발행 기술분산 장부를 사용해 자산을 등록하고 거래 기록으로 소유권 증명을 가능하게 한다.
자산 중심 기술자산의 디지털 표현과 관리에 초점을 맞추고, 프라이빗 네트워크를 통해 자산 거래를 용이하게 한다.
응용 플랫폼이더리움처럼 블록체인 기술을 플랫폼화해 다른 분야에 적용한다.
허가형 분산 원장프라이빗 블록체인처럼 기존 금융 시스템의 비용, 속도, 결제 완결성 문제를 개선하려는 기술이다.
SECTION 05

Web 3.0 보안 위협

Web 3.0의 보안 위협은 애플리케이션, 시스템, 네트워크 모든 부분에 존재한다. 하지만 Web 3.0이 블록체인에 많이 기대고 있으므로, 블록체인의 보안 위협을 Web 3.0 보안 위협으로 봐도 된다고 설명한다.

  • 애플리케이션에서는 DApp, 스마트 컨트랙트, 월렛 피해가 발생할 수 있다.
  • 시스템에서는 블록체인 코드 취약점이 전체 시스템에 영향을 줄 수 있다.
  • 네트워크에서는 인프라 구조적 취약점을 활용해 무단 채굴 등에 악용할 수 있다.
  • NFT 마켓플레이스와 이용자를 대상으로 한 악성 코드, 피싱, 지갑 탈취가 발생할 수 있다.
SECTION 06

DeFi와 보안 이슈

DeFi는 블록체인 네트워크에서 스마트 계약을 기반으로 가상자산을 이용해 동작하는 탈중앙화 금융 서비스다. 누구나 네트워크를 통해 금융 서비스를 자유롭게 이용할 수 있고, 금융 거래에 특정 운영 주체나 중개인의 개입이 필요하지 않다.

Web 3.0에서 디지털 가치는 사용자가 애플리케이션을 만들고 다른 사용자에게 직접 지불받을 수 있게 한다. 이때 중개자 필요성을 줄이는 데 DeFi가 중요한 역할을 한다. 기본 토큰은 비트코인이나 이더리움처럼 특정 블록체인 프로토콜에 연결되고, 사용자가 생성한 토큰도 같은 프로토콜을 활용해 쉽게 거래될 수 있다.

하지만 DeFi는 효율성에 비해 보안성이 낮아 해커들이 관심을 많이 가지는 분야다. 강의는 가상자산 범죄 피해액이 140억 달러 수준이며, 자금 도난 사건 중 큰 비중이 DeFi 플랫폼에서 발생한다고 설명한다. Wormhole 토큰 브릿지 해킹, 라자루스의 DeFi 플랫폼 이더리움 탈취, Steadefi 공격 사례가 언급된다.

DeFi 해킹은 주로 서비스 프로그래밍 코드상의 보안 취약점을 공격하거나 관리자 계정을 탈취하는 방식으로 이루어진다. 전문가들은 DeFi가 디지털 범죄의 새로운 온상이 되고 있다고 경고하며, 투자나 서비스 이용 시 주의해야 한다고 말한다.

SECTION 07

DAO와 The DAO 해킹

DAO는 사람이 아니라 스마트 계약에 인코딩되고 실행되는 프로토콜로 관리되는 조직이다. 조직 구조, 거버넌스, 활동의 주요 측면이 블록체인 스마트 계약으로 프로그래밍될 수 있다.

DAO는 작동 규칙을 스마트 컨트랙트로 인코딩한 뒤, 자금 조달 단계로 들어간다. 내부 재산으로 쓸 토큰이 필요하고, 사용자는 DAO에 투자함으로써 투표권을 갖고 운영 방식에 영향을 미친다. 배치 이후 DAO는 개발자를 포함한 누구에게도 독립적으로 작동하며, 오픈소스 코드와 블록체인 기록으로 투명성을 가진다.

문제는 초기 코드에 보안 허점이 있어도 과반 투표 전까지 수정이 어렵다는 점이다. The DAO는 2016년 독일 스타트업이 개발한 프로젝트로, 토큰 세일을 통해 1억 5천만 달러 이상을 조달한 성공적인 크라우드펀딩 사례였다. 하지만 코드에 버그가 있었고, 공격자는 환불 기능의 재귀 호출 문제를 이용해 이더를 받고도 DAO 토큰을 돌려주기 전에 다시 환불받는 무한 환불 공격을 만들었다. 결국 당시 가치로 약 750억 원이 탈취되었다.

흥미로운 점은 공격자가 엄밀히 따지면 코드에 적힌 규칙을 따라 행동했다는 것이다. 이 사건은 “코드는 법이다”라는 원칙과 보안 취약점 사이의 충돌을 보여 주었고, 이더리움과 DAO 개념 자체의 평판에도 큰 손상을 주었다.

SECTION 08

NFT와 보안 이슈

NFT는 같은 종류의 토큰이라도 각각 고유한 값을 가져 교환이 불가능한 토큰이다. 디지털 콘텐츠에 대한 정보를 담고 있고, 블록체인 거래 내역을 통해 진위 여부와 소유권을 증명할 수 있다. 구성 요소는 스마트 컨트랙트, 메타데이터, 디지털 콘텐츠다.

NFT 특징설명
신뢰성블록체인 기록을 통해 진위와 소유권을 확인할 수 있다.
대체 불가능성각 토큰이 고유하므로 서로 동일하게 교환되지 않는다.
소유권 증명디지털 콘텐츠의 소유권을 기록으로 증명한다.
고유성토큰마다 고유한 속성과 기록을 가진다.

NFT 보안 문제는 블록체인 자체 공격과 외부 연계 서비스 공격으로 나뉜다. 블록체인 자체 공격은 스마트 컨트랙트나 기반 소스코드 취약점을 악용해 암호자산이나 NFT를 탈취하는 방식이다. 외부 서비스 공격은 NFT 원본 데이터가 저장된 외부 저장소, 마켓플레이스, 지갑 관리 시스템, 커뮤니티 관리자 계정을 공격하는 방식이다.

NFT는 이더리움 기반에서 가스비와 블록 데이터 크기 문제 때문에 원본 데이터를 블록체인 밖에 두고 저장소 정보와 메타데이터만 기록하는 경우가 많았다. 해커는 외부 저장소를 해킹해 원본 데이터를 탈취하거나 복제하고, NFT에 기록된 저장소 정보를 변경하거나 다른 저장소로 강제 이동시키는 공격을 할 수 있다. 디스코드나 트위터 같은 커뮤니티 관리자 계정을 탈취해 이용자 지갑 정보를 빼내는 피싱·스미싱도 발생한다.

SECTION 09

암호화폐 해킹과 CBDC

암호화폐 해킹은 계속 발생한다. 강의에서는 중국계 거래소 ZB의 암호화폐 도난, 크로스체인 기업 Nomad 해킹, Solana 지갑 8천여 개 해킹 사례를 언급한다. 디지털 화폐 관련 해킹 사고는 전 세계적으로 끊이지 않는다.

이런 흐름 속에서 CBDC도 대두된다. CBDC는 전자적 형태로 발행되는 중앙은행 화폐다. 이용 목적에 따라 모든 경제주체의 일반 거래에 쓰이는 소액결제용 CBDC와 은행·금융기관 간 거래에 쓰이는 거액결제용 CBDC로 구분된다.

CBDC는 실물 명목화폐를 대체하거나 보완하기 위해 등장했고, 현금보다 거래 기록 추적이 쉬워 불법 자금 문제 완화에 기여할 수 있다. 구현 방식은 중앙은행 또는 시중은행이 계좌와 거래 정보를 관리하는 단일원장 방식, 그리고 다수 거래 참가자가 동일한 거래 기록을 관리하는 분산원장 방식으로 나뉜다.

강의는 Web 3.0, 블록체인, DeFi, DAO, NFT, CBDC 같은 키워드를 바탕으로 각자 관심 있는 분야를 더 깊게 공부하라고 마무리한다.
