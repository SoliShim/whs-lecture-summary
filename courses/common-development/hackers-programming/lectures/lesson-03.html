HTTP 요청과 응답

HTTP는 Hypertext Transfer Protocol의 약자이며, 웹브라우저와 웹서버가 데이터를 주고받기 위한 규칙이다. 웹브라우저는 클라이언트로서 요청을 보내고, 서버는 그 요청에 대한 응답을 돌려준다. 이 구조를 request와 response 방식이라고 이해하면 된다.

요청과 응답은 모두 헤더와 바디로 구성된다. 요청에는 GET, POST, PUT, DELETE 같은 메소드가 있고, 응답에는 200, 300, 400, 500 계열 상태 코드가 있다. 강의에서는 특히 요청 쪽에서는 GET과 POST, 응답 쪽에서는 정상 응답인 200과 클라이언트 오류 계열인 400번대를 우선 보면 된다고 설명한다.

구분강의에서 보는 핵심
GETURL과 쿼리 문자열에 필요한 인자를 붙여 서버에 데이터를 요청한다.
POST요청 바디에 데이터를 담아 서버에 보낸다. 폼 데이터나 JSON을 보낼 수 있다.
200요청이 정상 처리되었음을 의미한다.
400 계열잘못된 요청, 권한 문제, 존재하지 않는 페이지처럼 클라이언트 쪽 문제가 있음을 나타낸다.

예시 요청과 실제 웹사이트 헤더

예를 들어 브라우저가 www.example.com의 index.html을 요청한다면, 요청에는 HTTP 버전, User-Agent, Accept-Encoding 같은 정보가 들어간다. 서버는 HTTP 1.1 200 OK 같은 상태 줄과 함께 Content-Type, 문자 인코딩, Content-Length 같은 정보를 돌려주고, 바디에는 실제 HTML이 들어간다.

강의에서는 화이트햇 스쿨 홈페이지에 직접 요청을 보내 본 결과도 보여 준다. 실제 요청은 예시보다 훨씬 많은 헤더를 포함한다. 쿠키, 세션, Accept-Encoding, Sec-로 시작하는 브라우저 보안 관련 헤더들이 붙는다. 응답 역시 HTML을 제외해도 많은 헤더를 포함하며, X-Frame-Options, X-XSS-Protection 같은 보안 관련 헤더가 보인다. 핵심 구조는 예시와 같지만 실제 웹은 상태 유지와 보안을 위해 더 많은 정보를 주고받는다.

HTML, CSS, JavaScript

HTTP로 받아오는 바디에는 대개 HTML 코드가 들어 있다. HTML은 Hypertext Markup Language의 약자이며, 웹페이지를 구조화하고 콘텐츠를 표시하는 표준 마크업 언어다. 여는 태그와 닫는 태그가 있고, head, body, p, a, div 같은 태그가 계층적으로 중첩된다.

웹페이지는 HTML만으로 완성되지 않는다. HTML은 구조를 담당하고, CSS는 글자 모양·색상·레이아웃 같은 꾸밈을 담당하며, JavaScript는 버튼 클릭, 광고 배너, 동적 표시처럼 기능적 동작을 담당한다. 그래서 웹 스크래핑을 하려면 화면에 보이는 결과뿐 아니라 그 화면을 만든 HTML 구조를 볼 수 있어야 한다.

소스 보기와 개발자 도구

브라우저에서 마우스 오른쪽 클릭 후 페이지 소스 보기를 누르면 HTML 코드를 볼 수 있다. 처음 보면 복잡해 보이지만, 태그들이 계층을 이루고 있고 화면에 보이는 요소들이 어떤 HTML 구조에 대응하는지 추적할 수 있다.

특정 글자나 버튼이 HTML 코드 어디에 있는지 찾기 어려울 때는 Chrome 개발자 도구를 사용한다. F12, Ctrl + Shift + I, 또는 마우스 오른쪽 클릭 후 검사를 누르면 왼쪽 화면과 오른쪽 HTML 요소가 1대1로 대응된다. 코드 일부를 바꾸면 왼쪽 화면이 어떻게 바뀌는지도 바로 확인할 수 있다. 이 과정은 나중에 BeautifulSoup으로 특정 태그를 선택할 때 기준을 잡는 데도 중요하다.

requests 패키지와 GET/POST

Python의 requests 패키지는 HTTP 요청을 쉽게 보내고 응답을 객체로 받아오게 해 준다. 설치는 터미널에서 pip install requests로 할 수 있다. 이후 프로젝트가 커지면 virtualenv나 Poetry 같은 가상 환경 도구로 라이브러리 버전을 관리하는 것이 좋다.

GET 요청에서는 URL 뒤의 쿼리 문자열에 들어갈 인자를 params 딕셔너리로 넘긴다. POST 요청에서는 본문에 들어갈 데이터를 data 또는 json 인자로 넘긴다. 폼 전송처럼 a=b&c=d 형태로 보내는 경우는 data, JSON 객체를 보내는 경우는 json을 사용한다.

1번부터 100번까지 자동으로 찾기

실습 사이트에는 1번부터 100번까지의 노트가 있고, 그중 한 페이지에만 WHS 문자열이 포함되어 있다. 수동으로 100번 클릭하는 대신 URL의 숫자 부분이 규칙적이라는 점을 이용해 반복 요청을 보낸다.

처음 requests.get() 결과를 그대로 출력하면 <Response [200]>처럼 상태만 보인다. 실제 HTML을 보려면 response.text나 response.content.decode()를 사용한다. 한글이 깨지는 경우에는 바이트로 받은 뒤 적절히 디코딩하면 된다.

반복문으로 1부터 100까지 URL을 만들고, 응답 본문 안에 WHS가 있을 때만 출력한다. 강의에서는 73번 노트에서 congratulations the flag is WHS simple web scraping이라는 결과를 찾는 흐름을 보여 준다. 반복 진행 상황을 보기 위해 tqdm으로 루프를 감싸는 방법도 소개한다.

import requests
from tqdm import tqdm

for idx in tqdm(range(1, 101)):
    url = f"https://note.nyan101.com/{idx}"
    response = requests.get(url)
    body = response.content.decode()

    if "WHS" in body:
        print(idx)
        print(body)
