SECTION 01

Python과 VS Code를 쓰는 이유

강의는 Python 3와 Visual Studio Code를 실습 도구로 사용한다. Python은 수치해석, 공학, 과학, CTF, AI 분야에서 널리 쓰이는 범용 프로그래밍 언어다. CTF에서는 pwntools 같은 라이브러리가 자주 쓰이고, AI에서는 PyTorch와 TensorFlow 같은 라이브러리가 Python 생태계에서 널리 사용된다. 대학의 첫 프로그래밍 수업에서도 많이 쓰일 정도로 입문 난도가 낮은 편이다.

VS Code는 Python 전용 도구는 아니지만 다양한 확장 기능을 설치할 수 있고 무료로 사용할 수 있어 실습 환경 구성에 적합하다. 이 과목에서는 Python 자체와 VS Code를 연결하고, 코드를 빠르게 실행할 수 있도록 Code Runner 확장 기능을 사용한다.

SECTION 03

Base64가 필요한 이유

Base64는 CTF나 프로그래밍에서 자주 마주치는 인코딩이다. 흔히 끝에 = 문자가 있으면 Base64일 가능성이 있다고 말하지만, 강의에서는 이것을 농담 섞인 첫 힌트로 소개하고 실제 원리를 설명한다.

컴퓨터의 한 바이트는 0x00부터 0xFF까지 256가지 값을 가질 수 있다. 하지만 사람이 화면에 그대로 출력해서 복사·붙여넣기할 수 있는 ASCII 문자는 훨씬 적다. 모든 바이트 값을 안전하게 문자 형태로 표현하려면 출력 가능한 문자 집합으로 바꾸는 방식이 필요하다.

16진수로 표현하면 한 바이트를 00, 01, FF처럼 두 글자로 나타낼 수 있지만, 한 바이트를 표현하는 데 두 바이트가 필요해 약 두 배의 공간을 쓴다. Base64는 대문자 26개, 소문자 26개, 숫자 10개, +, /를 합쳐 총 64개의 문자로 데이터를 표현한다. 64는 2의 6제곱이므로 6비트마다 한 글자로 바꿀 수 있고, 이 방식은 16진수보다 공간 낭비가 적다.

표현 방식특징
원본 바이트모든 값이 화면에 안전하게 출력되지 않는다.
16진수한 바이트를 두 글자로 표현하므로 이해는 쉽지만 길이가 커진다.
Base6464개 문자로 6비트 단위 표현을 하며, 끝자리가 맞지 않을 때 = 패딩을 붙인다.

한글, 일본어, 중국어 같은 문자는 Unicode, UTF, CP949 등 별도 문자 인코딩과 관련된 문제다. Base64는 그런 문자 자체를 정의하는 규칙이 아니라, 이미 존재하는 바이트열을 출력 가능한 문자로 바꾸는 범용 표현 방식으로 이해해야 한다.

SECTION 04

CyberChef로 먼저 확인하기

CyberChef는 영국 정보기관 GCHQ가 공개한 오픈소스 분석 도구다. 강의에서는 “사이버 스위스 아미 나이프”처럼 다양한 변환과 분석을 한 화면에서 조합할 수 있는 도구로 소개한다. Base64 인코딩·디코딩, Hex 변환, AES·DES 같은 암호화, 해시, HTTP 요청, 문자열 처리 등 많은 기능을 레시피처럼 끌어다 쓸 수 있다.

CyberChef의 기본 사용 흐름은 왼쪽의 연산 목록에서 필요한 작업을 가운데 레시피 영역으로 끌어오고, 오른쪽 위 입력 창에 데이터를 넣은 뒤, 오른쪽 아래 출력 결과를 확인하는 방식이다. 강의에서는 문자열을 Base64로 바꾸고 다시 디코딩하는 작업, Hex 변환, AES 암호화 예시까지 가볍게 보여 준다. AES는 키와 IV 길이가 맞아야 오류가 사라진다는 점도 함께 확인한다.

SECTION 05

Base64 문자열 1만 개에서 필요한 값 찾기

실습 파일에는 Base64로 인코딩된 문자열이 한 줄에 하나씩 약 1만 개 들어 있고, 그중 하나만 WHS로 시작하는 값을 디코딩 결과로 갖는다. CyberChef에 한 줄씩 붙여 넣어 확인할 수도 있지만, 1만 줄을 수작업으로 검사하는 것은 비효율적이다. 그래서 Python으로 파일을 읽고, 각 줄을 Base64 디코딩한 뒤 조건을 검사한다.

먼저 open()과 readlines()로 파일의 각 줄을 읽는다. 단순히 출력하면 보기 불편하므로 pprint로 정돈된 출력도 확인한다. Python에는 base64 패키지가 기본 제공되며, base64.b64decode()로 디코딩할 수 있다.

디코딩 결과 앞에 b가 붙어 보이면 그것은 문자열이 아니라 bytes 타입이라는 뜻이다. 바이트는 일반 문자뿐 아니라 널 바이트, 공백 문자, 출력되지 않는 데이터까지 포함하는 자료형이다. 사람이 읽는 문자열로 다루려면 .decode()를 호출해 str 타입으로 바꾼다. 마지막으로 startswith("WHS") 조건을 걸어 원하는 값만 출력한다.

import base64
from pprint import pprint

with open("base64examples.txt", "r", encoding="utf-8") as file:
    lines = file.readlines()

for line in lines:
    decoded = base64.b64decode(line.strip()).decode()
    if decoded.startswith("WHS"):
        print(decoded)

Q. 파이썬어 3 프로그래밍 언어와 비주얼 스튜디오 코드라는 편집기를 사용할 건데요. 파이썬어 같은 경우에는 오늘날 가장 널리 쓰이는 프로그래밍 언어고 저희가 수치해석이라든지 공학, 과학 혹은 CTF대회에 자주 쓰이는 폰 라이브러리 그리고 또 AI에 자주 쓰이는 파이터치라든지 텐서플로우드 다양한 라이브러리를 제공하는 굉장히 범용적인 프로그래밍 언어입니다. 그리고 이제 대학에서 첫 수업으로 가장 많이 쓰일 정도로 난이도가 조금 낮은 편에 독하는 그런 언어라고 할 수 있겠습니다. 비주얼 스튜디오 코드 같은 경우는 저희가 파이썬어 전용으로 만들어진 파이썬이라든지 다른 편집기랑 조금 낮은 게 저희가 다양한 확장 기능을 설치해서 쓸 수 있고 무엇보다도 무료라는 점 때문에 저희가 비주얼 스튜디오 코드를 이용해서 이런 실습 환경을 구축해보겠습니다. 먼저 파이썬 설치하는 것에 대해 설명드리겠습니다. 파이썬어 공식 웹사이트를 통해 들어갈 수 있는데요. 이제 파이썬.org에 들어가시면 가장 먼저 메인 화면에 다운로드라는 페이지에 파이썬어 3.13.2라는 게 있을 겁니다.

이걸 누르시면 설치 파일을 다운받게 되고 이걸 실행하시게 되면 그냥 기본적으로 설치가 이루어지게 됩니다. 설치하실 때 주의하실 점 중에 아래를 보시면은 Add Python EXE to Pass 같은 항목이 있는데요. 기본 설정으로는 체크가 아마 해제되어 있을 텐데 체크를 하고 넘어가겠습니다. 이렇게 함으로써 나중에 있을 비주얼 스튜디오 코드와 연동을 좀 더 쉽게 하고 나중에 휠습에 있어서 조금 더 간편하게 할 수 있는 그런 설정이라고 보시면 되겠습니다. 다음으로 비주얼 스튜디오 코드 같은 경우에는 마찬가지로 이제 공식 페이지 code.visualstudio.com에 들어가셔서 메인 페이지에 바로 나오는 다운로드 폴 윈도우 버튼을 누르시면 되고 맥이나 다른 걸 쓰시는 분들은 다른 버튼이라고 할 텐데 그냥 메인 화면에 있을 겁니다. 이걸 통해 설치하실 때 아래 4개 항목을 모두 체크해 주시면 되겠습니다. 이것들 역시도 이제 이후에 실습을 조금 편하게 하고 저희가 나중에 혹여나 문제가 생길 때 조금 더 이제 디버깅을 좀 편하게 하는 그런 설정이라고 보시면 되겠습니다.
