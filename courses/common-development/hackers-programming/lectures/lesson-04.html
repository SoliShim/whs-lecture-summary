SECTION 01

API의 의미

API는 Application Programming Interface의 약자다. 서로 다른 소프트웨어 애플리케이션이 상호작용하기 위해 정해 둔 인터페이스다. “이 요청을 이런 형식으로 보내면 이런 데이터를 돌려주겠다”는 약속이라고 이해하면 된다.

API에는 운영체제의 POSIX API, 데이터베이스 API, 모바일 SDK API, 웹서비스 업체가 제공하는 자체 API 등 여러 종류가 있다. 이 강의에서는 HTTP 또는 HTTPS로 요청과 응답을 주고받는 웹 API에 집중한다. 클라이언트가 HTTP 프로토콜로 요청을 보내면 서버는 사전에 정한 형식에 따라 데이터를 처리하고 반환한다.

SECTION 02

브라우저 검색과 API 검색의 차이

강의는 네이버 뉴스를 웹브라우저로 검색했을 때와 검색 API로 호출했을 때를 비교한다. 브라우저 검색 요청에는 사용자가 직접 신경 쓰기 어려운 많은 인자와 헤더가 붙고, 응답에는 검색 결과뿐 아니라 CSS, JavaScript, 버튼, 레이아웃, 광고 등 화면을 꾸미기 위한 요소가 많이 섞인다. 자동화하려면 이 복잡한 HTML에서 필요한 데이터만 다시 파싱해야 한다.

반면 검색 API는 내가 어떤 인자로 검색하는지 명확하고, 응답도 대개 JSON처럼 프로그램이 바로 읽기 좋은 형태로 온다. 검색 결과의 제목, 링크, 요약, 작성 시각 같은 값이 구조화되어 있기 때문에 코드가 간결하고 로직이 분명해진다.

방식특징자동화 관점
브라우저 화면사람이 보기 좋게 디자인된 HTML, CSS, JavaScript가 함께 온다.필요한 값을 따로 찾아 파싱해야 한다.
웹 API정해진 요청 인자와 JSON 같은 구조화된 응답을 제공한다.프로그램이 바로 처리하기 쉽다.
SECTION 03

API 제공 분야

API는 여러 서비스에서 제공된다. 일반 포털은 검색, 캘린더, 문서, AI 기능을 API로 제공할 수 있고, 보안 분야에서는 CTI 관련 API가 중요하다. Criminal IP, Shodan, VirusTotal 같은 서비스는 IP 위협 점수, 취약점 정보, 악성코드 관련 정보, 다른 회원사에서 관찰된 활동 같은 보안 데이터를 제공한다. 무료 버전과 유료 버전이 나뉘며, 유료 버전이 더 많은 기능과 한도를 제공하는 경우가 많다.

메신저 API도 있다. Telegram, Slack, Discord 같은 서비스는 봇을 만들어 메시지를 보내거나 특정 명령에 반응하게 하는 기능을 제공한다. AI API는 ChatGPT나 Claude 같은 모델에 프롬프트를 보내고 결과만 받아와 자체 서비스에 붙이는 데 쓰인다. 클라우드 API는 AWS, Azure, GCP 같은 클라우드 자원을 콘솔 클릭 없이 코드로 관리하기 위해 사용되며, 이때는 API 키보다 액세스 키라는 표현을 많이 쓴다.

SECTION 04

API 키 유출 사고

API를 사용하는 쪽에서 가장 조심해야 할 것은 키 관리다. 공개되는 소스 코드나 설정 파일에 API 키를 그대로 넣어 GitHub에 올리면, 다른 사람이 그 키를 가져가 비용이 발생하는 API를 마음대로 사용할 수 있다. 데이터베이스 접속 정보나 AWS 액세스 키가 노출되면 더 큰 침해로 이어질 수 있다.

한 번 공개된 키는 여러 곳으로 퍼질 수 있다. 공개 저장소, 블로그 글, 설정 파일, 이후 수집된 LLM 학습 데이터셋까지 흘러갈 수 있다. 또 최근에는 오류가 난 코드를 그대로 AI 대화창에 붙여 넣는 경우가 많다. 그 코드 안에 비밀키, 로그, 설정값이 포함되어 있으면 AI 서비스 자체가 침해되거나 대화 내용이 노출될 때 함께 새어 나갈 수 있다.

SECTION 05

키 관리 원칙과 실습 방향

가장 기본 원칙은 API 키를 소스 코드에 하드코딩하지 않는 것이다. 별도 설정 파일이나 환경 변수로 분리하고, 그 파일은 공개 저장소나 클라우드 공유 폴더에 올리지 않는다. 사용량 기반으로 비용이 청구되는 API라면 정기적으로 사용량을 확인해 내가 쓰지 않은 호출이 있는지 살펴야 한다.

유출이 의심되거나 사용 이력이 이상하면 키를 즉시 만료하고 새로 발급한다. 비밀번호가 유출되면 바꾸는 것과 같은 대응이다. 강의에서는 별도 문제를 제공하지 않지만, 네이버 검색 API는 네이버 계정으로 무료 발급이 가능하고 한도도 비교적 넉넉하므로, 키워드를 입력받아 블로그나 뉴스 글을 검색하는 Python 스크립트를 작성해 보라고 안내한다.

import os
import requests

keyword = input("검색어: ")
headers = {
    "X-Naver-Client-Id": os.environ["NAVER_CLIENT_ID"],
    "X-Naver-Client-Secret": os.environ["NAVER_CLIENT_SECRET"],
}
params = {"query": keyword, "display": 5}

response = requests.get(
    "https://openapi.naver.com/v1/search/news.json",
    headers=headers,
    params=params,
)
response.raise_for_status()

for item in response.json()["items"]:
    print(item["title"], item["link"])

이번 시간에는 API란 무엇인지 우리가 API를 이용해서 어떤 것들을 할 수 있는지에 대해서 간단히 소개해보는 시간을 가지겠습니다. API는 어플리케이션 프로그래밍 인터페이스 라는 말처럼 서로 다른 소프트웨어 어플리케이션 간의 상호작용을 적용한 인터페이스로 어떤 소프트웨어인지는 명시하지 않고 그냥 서로 다른 것들 간에 이걸 쓰면 통신이 된다 내가 이걸 주면 내가 이런 데이터를 주겠다라는 걸 약속한 인터페이스라고 보시면 되겠습니다. 이런 API의 예로는 운영체제에 쓰이는 포직스 API 아니면 데이터베이스나 모바일 SDK에 있어서 API 아니면 웹서비스 업체에서 제공하는 자체 API 등 굉장히 많은 종류들이 있는데요. 이번 강의에서는 웹 API에 대해서 한번 집중하겠습니다. 웹 API는 크게 클라이언트가 API를 통해 인터넷에 있는 어딘가의 서버 요청을 하면 이 서버에서 자체적인 기능들을 통해서 데이터를 처리하고 그 데이터를 반환하는 그런 형식이라고 보시면 되겠습니다. 구체적으로는 이제 서비스 업체에서 HTTP 혹은 HTTPS 형태로 데이터 교환 기능을 위해 제공하는 API 웹 API라고 말하고 이건 이제 클라이언트 쪽에서 HTTP 프로토콜을 이용해서 요청을 보내면 서버에서 사전에 약속된 정의에 따라서 이 데이터에 대한 응답을 제공해주는 그런 거라 보시면 되겠습니다.

이런 API는 다양한 서비스와 연동해서 나만의 코드를 작성할 수 있고 나만의 코드에 대해서 추가적인 기능을 활용할 수 있는 그런 역할을 하고 있습니다. 그럼 이런 API를 왜 쓰는가 어차피 HTTP면은 웹 브라우저 쓰는 거랑 똑같지 않나요? 많은 말을 할 수 있습니다. 저희가 이제 네이버 검색에 대해서 웹 브라우저를 사용할 때와 네이버 검색 API를 사용했을 때 두 경우에 대해서 요청과 이제 응답을 비교해보겠습니다. 먼저 이건 저희가 웹 브라우저를 통해서 네이버 뉴스를 검색했을 때 들어가는 왼쪽 요청과 그에 대한 리스폰스입니다. 보시면 저희가 이제 각 수많은 인자들 중에서 각 인자가 어떤 역할을 하는지 그리고 내가 구체적으로 어떤 인자를 조작해서 보내야 되는지에 대해서 약간 불명한 부분이 있고 그 다음 저도 모르는 역할에 수많은 헤너들이 있습니다. 응답에 대해서도 굉장히 나는 검색 결과만 깔끔하게 정리하는 걸 보고 싶은데 굉장히 많은 디자인적 요소를 위한 CSS 버튼을 꾸미기 위한 자바스크립트 그 다음 수많은 뉴스의 제목과 요약된 내용 그 다음 뉴스의 작성 유시들이 HTML 태그 안에 남겨있죠.

이것도 굉장히 어찌 보면 정돈되었음이라도 살짝 난잡하게 이런 결과를 받아볼 수 있습니다. 물론 이제 이거를 저희가 앞서 배운 BS4라던지 이런 패키지를 통해서 활싱할 수도 있는데 API 사용의 결과를 보겠습니다. 이건 이제 한눈에 봐도 이전과는 굉장히 깔끔하게 됐죠. 요청 자체도 내가 무슨 인자에 대해서 검색을 하는지 굉장히 명료하게 되어 있고 들어오는 결과도 이제 굳이 내가 파싱을 하지 않아도 될 정도로 JSON으로 굉장히 깔끔하게 정돈된 결과물이 나오는 걸 알 수 있습니다. 따라서 이제 이런 API를 사용하게 될 경우에는 저희가 굉장히 코드를 작성함에 있어서 굉장히 간결하고 명료하고 정돈된 로직을 작성할 수 있다 이런 장점이 있다 보시면 되겠습니다. 그럼 이런 API를 제공하는 업체들은 주로 어떤 것들을 API로 제공하는지에 대해서 살펴보겠습니다. 가장 먼저 일반 서비스 혹은 검색이라던지 블로그 자체의 서비스 업체에 제공하는 기능들을 제공하는 게 있는데요. 주로 네이버라던지 구글 이런 종합 포털을 제공합니다. 네이버 같은 경우에는 아래 보이는 화면처럼 간단하게 네이버 검색, 네이버 클로버 AI, 캡차 네이버 캘린더에 대한 일정 권리 등을 이제 자체 API로 제공을 하게 되고 구글도 마찬가지로 검색, DNS 혹은 네이버 구글 캘린더와의 연동 혹은 구글 시트와의 연동 이런 것들 다양한 기능들을 자체 API로 제공하고 있습니다.
