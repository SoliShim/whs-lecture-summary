SECTION 01

n8n의 역할

n8n은 오픈소스 워크플로우 자동화 도구다. 코드를 직접 많이 작성하지 않아도 그래프 형태로 주요 작업 흐름을 시각적으로 구성할 수 있다. IT 인프라 작업, DevSecOps 업무 자동화, 개인 일정·다이어리 자동화, 소규모 비즈니스 운영까지 폭넓게 활용할 수 있다.

가장 큰 특징은 드래그 앤 드롭 기반으로 노드를 연결하는 직관적인 UI다. Telegram, Discord, Slack 같은 메신저, MySQL과 PostgreSQL 같은 데이터베이스, Gmail, Google Docs, AWS, Azure 같은 클라우드 서비스까지 매우 많은 애플리케이션과 연동할 수 있다.

SECTION 02

활용 예시
상황자동화 흐름
소규모 쇼핑몰 메일 확인들어온 메일을 검사하고 피싱으로 의심되면 Slack으로 알림을 보낸다.
업무 데이터 동기화PostgreSQL에 저장된 정보를 Google Sheets에도 업데이트하고, 특별한 정보가 있으면 Slack 알림을 보낸다.
보안 로그 감시서버 로그인 로그를 실시간으로 받고, 처음 보는 IP를 VirusTotal에 조회한 뒤 악성·의심 결과가 나오면 Slack과 Gmail로 알린다.

강의는 n8n을 지금까지 배운 HTTP 요청, API, 데이터 처리 흐름을 24시간 동작하는 자동화로 이어 주는 도구로 소개한다.

SECTION 03

설치와 사용 방식

n8n을 사용하는 방법은 크게 두 가지다. 첫 번째는 공식 사이트에서 계정을 만들고 서비스가 제공하는 환경을 사용하는 것이다. 별도 설치가 필요하지 않고, 이메일 가입 시 약 14일 무료 체험을 제공하므로 먼저 기능을 알아보기 좋다.

두 번째는 Docker 이미지로 로컬에 구축하는 방식이다. n8n은 오픈소스 도구이고 공식 Docker 이미지를 제공한다. Docker Desktop 같은 도구로 이미지를 내려받아 내 PC 안에 n8n 서버를 띄우면, 서버가 돌아가는 동안 별도 유료 회원 제한 없이 기능을 사용할 수 있다. 개인이 가볍게 써 볼 때는 공식 서비스가 편하고, 본격적으로 운영해 보고 싶다면 Docker 기반 로컬 구축이 적합하다.

SECTION 04

노드 종류와 흐름

n8n의 핵심은 노드들을 그래프로 연결하는 것이다. 트리거 노드에서 흐름이 시작되고, 데이터는 웹 요청, 중복 제거, 코드 실행, Google Sheets, Slack 같은 노드를 지나며 변환되고 전달된다.

SECTION 05

디버깅과 네이버 검색 API 예시

n8n에서 각 노드를 클릭하면 그 노드를 거치기 전과 후의 데이터가 표시된다. 어떤 입력이 들어왔고, 노드가 어떤 출력을 만들었는지 볼 수 있으므로 디버깅이 쉬워진다.

강의에서는 네이버 검색 API를 쓰는 노드를 예로 든다. 왼쪽 입력에는 해킹 같은 키워드가 들어오고, HTTP Request 노드는 네이버 검색 API를 호출한다. 결과는 오른쪽 출력에서 JSON 형태로 깔끔하게 정리되어 다음 노드로 넘어간다. 이전 강의에서 Python으로 한 번 검색을 실행했다면, n8n에서는 그 실행을 스케줄링하고 다른 서비스와 연결할 수 있다.

Schedule Trigger: 매일 아침 실행
-> Set: keyword = "보안", "사이버", "해킹"
-> HTTP Request: 네이버 검색 API 호출
-> Code or Item Lists: 필요한 제목, 링크, 요약만 정리
-> Google Sheets: 검색 결과 행 추가
-> Slack or Gmail: 새 결과가 있으면 알림

강의에서는 모든 수강생이 n8n 환경을 개별 구축하기 어려울 수 있어 별도 필수 실습으로 두지는 않는다. 대신 관심이 있다면 네이버 검색 API 키를 이용해 매일 아침 보안, 사이버, 해킹 키워드로 뉴스를 검색하고 그 결과를 Google Sheets에 자동 정리하는 워크플로우를 만들어 보라고 권한다.

N8N, 혹은 네이트은이라고도 발음하는데요. 이건 오픈서스로 되어 있는 워크플로우 자동화 도구로 굉장히 많은 어플리케이션과 서비스 간의 통합을 지원하고 있습니다. 가장 큰 특징으로는 코드가 직접 짜지 않고도 아래 보이는 것처럼 그래프 형태로 시각적으로 주요. 작업 후렴을 구성할 수가 있고 이걸 이용해서 기본적인 IT 인프라 작업, 아니면 이제 데브색옵스라고도 하죠. 이런 업무 자동화라든지 개인, 다이어리 업무의 자동화 아니면 소규모 비즈니스까지 가능할 정도로 굉장히 폭넓은 기능을 가지고 있는 도구라고 보시면 되겠습니다. N8N의 가장 큰 특징으로는 이제 보이는 것처럼 그래프를 드래그 앤 드롭 기반으로 모드끼리 연결함으로써 굉장히 직관적인 UI를 가지고 있다는 점과 다양한 애플리케이션과 연동시원 아래 보이는 것처럼 당연히 텔레그램, 디스코드, 슬랙 MySQL이나 포스그레임, 이런 데이터베이스 메신저와 연동이 있는 한편 메일 보내는 지메일, 아니면 구글독스, 아니면 이제 아마존 AWS라든지 MS Azure, 이런 클라우드에 이르기까지 굉장히 많은 총 거의 천여개에 이르는 애플리케이션과의 연동 기능을 지원하고 있습니다.

이런 N8N에 대해서 활용하는 예시를 보시면 이제 보이는 것처럼 소규모 비즈니스, 내가 이제 작은 쇼핑몰을 운영하는데 내가 메일이 왔다. 그럼 이 메일이 왠지 피싱인 것 같다. 이런 경우에 대해서, 들어온 메일에 대해서 의심 여부를 한번, 피싱 메일인 경우에 슬랙으로 알림을 주는 이런 플로우를 하나 만들어볼 수 있고 혹은 이제 IT 업무에 대해서, 들어온 거에 대해서 내가 이제 포스그레임, DB에 저장을 하겠지만 이 구글 시티에도 같이 업데이트를 해줬으면 좋겠다. 그리고 이 구글 시티 업데이트할 때 만약에 이게 새로운 정보나 특별한 정보라고 하면은 그냥 그대로 멈추지 말고 슬랙으로도 알림을 한번 더 줬으면 좋겠다. 이런 업무까지 자동화할 수가 있고 저희가 관심을 가지고 있는 사이버보안에 대해서는 서버에 대해서 들어오는 로그인을 실시간으로 전송을 하고 신규 모르는 IP가 들어왔을 때 얘를 바이러스 토탈 쪽에 한번 던진 다음 이 바이러스 토탈에 대한 결과가 악성이라든지 의심으로 나올 경우에 슬랙과 지메일에 알림을 주는 이런 식의 워크플로우를 만들어 볼 수도 있겠습니다.

이런 N8N을 설치하는 방법에 대해서는 두 가지 방법이 있는데요. 첫 번째 방법은 조금 더 쉬운 예시로 공식 사이트에 대해서 계정을 만들고 사이트에서 제공하는 시스템을 사용하는 겁니다. 이건 이제 별도의 설치 과정이 필요하지도 않고 당연히 이제 유럴 라이전스이긴 한데 이메일을 통해서 가입한 경우에는 약 14일간의 무료 체험기간을 제공하기 때문에 제가 N8N이 먼저 한번 알아보고 싶다. 라고 하면 이 방법을 추천드립니다. 다른 방법으로는 도커 이미지를 이용한 로컬 구축이 있겠습니다. 앞서서 N8N 자체가 오픈소스 도구라는 말을 했었는데요. 그런 만큼 이제 공식 사이트에서 공식적으로 이제 N8N에 대한 도커 이미지를 제공하고 있습니다. 이제 본인 PC의 도커 데스크탑이라든지 여러 가지 도구를 이용해서 도커 이미지를 받고 이걸 이용해서 본인 PC 안에 있는 자신만의 N8N 서버를 구축할 수가 있는데요. 이런 경우에는 특별한 과금이라든지 유료 회원 가입 없이도 이 서버가 구동하고 있는 한 별도의 제한 없이 N8N의 모든 기능을 자유롭게 사용할 수 있습니다.

먼저 그냥 개인이 사용하기 편한 건 첫 번째 방법이고 또 사용해보고 아 난 이걸 한번 시스템에 본격적으로 사용해봐야겠다 싶으면 도컬을 통한 두 번째 방법을 추천드립니다. 그럼 N8N의 기초적인 사용법에 대해 알아보겠습니다. N8N의 가장 큰 특징 가장 핵심적인 기능이라고 하면 이런 노드들을 그래프로 와닿아 내는 점인데요. 왼쪽에 보이는 것처럼 트리거 노드를 시작으로 해서 데이터가 이 노드를 따라서 흐름을 흐르게 되고 데이터가 중간에 있는 웹 요청기라든지 중복 제거 코드를 통해서 자바스크립트라든지 파이썬을 통한 세부적인 로직 슬랙이라든지 구글 시트에 대한 연동 이런 식으로 아래와 전체의 흐름을 하나의 그래프로 와닿아 내는 걸 목표로 하겠습니다. 그럼 이제 세부적인 노드들의 역할과 종류에 대해 알아보겠습니다. 트리거 노드라는 게 있습니다. 이 트리거 노드는 워크플로우에서 좀 특별한 역할을 하는 노드인데요. 전체 작업 흐름을 언제 어떤 조건을 만족했을 때 시작할 것인가에 대한 그 조건을 부여하는 노드라고 보시면 되겠습니다. 왼쪽에 보이는 것처럼 수동으로 실행하는 방법이 있을 거고 주기적으로 주기적으로 5분에 한 번 1분에 한 번 하루에 한 번이라든지 주말 아침마다 이런 게 될 수가 있겠죠.
