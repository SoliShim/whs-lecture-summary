SECTION 01

설치 대상

강의는 개발환경 구축을 Unix 환경과 Node.js LTS 설치에서 시작한다. 실습은 Ubuntu, macOS, WSL 같은 Unix 계열 터미널 흐름을 기준으로 진행된다. Node.js는 최신 기능만 따라가기보다 안정적으로 유지되는 LTS 버전을 설치하는 편이 실습과 운영에 적합하다.

설치 후에는 터미널에서 버전을 확인한다. `node`는 JavaScript 실행기이고, `npm`은 패키지 설치와 프로젝트 스크립트 실행을 담당한다.

node -v
npm -v
SECTION 02

npm의 역할

npm은 Node Package Manager의 약자로, 외부 라이브러리를 프로젝트에 설치하고 버전을 기록하며 실행 스크립트를 관리한다. Express, EJS, nodemon 같은 도구도 npm을 통해 설치한다.

명령의미
npm init -y기본 설정으로 package.json을 만든다.
npm install expressExpress를 dependencies에 설치한다.
npm install -D nodemon개발용 도구로 nodemon을 설치한다.
npm run devpackage.json에 등록한 dev 스크립트를 실행한다.
SECTION 03

VS Code와 실습 도구

VS Code는 파일 탐색기, 터미널, 코드 편집, 확장 설치를 한 화면에서 처리할 수 있어 Node.js 실습에 편하다. 강의에서는 VS Code를 설치하고, 필요한 확장과 터미널을 준비한 뒤 프로젝트 폴더를 열어 실습한다. PortSwigger 같은 보안 학습 사이트도 함께 열어 이후 웹 보안 흐름과 연결할 준비를 한다.

실습 환경에서 중요한 점은 명령을 실행하는 현재 위치다. `npm init`, `npm install`, `node index.js`는 프로젝트 폴더에서 실행해야 package.json과 node_modules가 올바른 위치에 생성된다.

SECTION 04

프로젝트 생성

프로젝트는 폴더 생성, 초기화, 패키지 설치 순서로 만든다. 설치가 끝나면 `package.json`, `package-lock.json`, `node_modules`가 생긴다. `package.json`은 프로젝트 이름, 버전, 실행 스크립트, 의존성 목록을 담는 핵심 파일이다.

mkdir web-practice
cd web-practice
npm init -y
npm install express
touch index.js
node index.js

WSL을 설치하기 위해서는 윈도우즈의 이제 기능 켜기 뜨기라는 제어판 기능 중에서 여기 보면은 이제 리눅스용 윈도우즈 하위 시스템이라는 게 있습니다. 이게 켜져야 리눅스 WSL을 윈도우에서 사용할 수 있어요. 이걸 체크하시고 이제 확인을 누르시면 이제 윈도우 리눅스 하위 시스템이 설치되면서 재부팅이 한번 될 거예요. 재부팅을 하고 나서는 이제 우분투를 마이크로소프트의 서프트 스토어에서 검색을 해 주시고 우분투 원하는 버전들을 이제 설치를 해 주시면 됩니다. 저는 이제 집에서 사용하고 있는 우분투 24.04 LTS 버전을 설치를 해 놨어요. 이걸 이제 설치를 하시면 금방 설치가 될 거예요. 이렇게 네 우분투 환경이 나옵니다. 네 정상적으로 우분투 환경이 나오죠. 그 다음에 이제 비주얼 스튜디오 코드를 설치를 할 건데요. 비주얼 스튜디오 코드는 이제 제가 사용한 에디터인데 실습 때 사용하는 에디터인데 사용하고 계신 에디터가 있으시면 그걸 쭉 사용하셔도 됩니다. 혹시 에디터가 없으신 분들을 위해서 실습 환경과 똑같은 비주얼 스튜디오 코드 사용을 추천드릴게요.
