SECTION 01

템플릿 엔진의 역할

템플릿 엔진은 서버의 데이터와 HTML 구조를 결합해 최종 HTML을 만든다. EJS는 Embedded JavaScript Template의 약자로, HTML 안에 JavaScript 표현식과 제어문을 넣을 수 있다. Express에서는 `res.render()`를 호출하면 EJS 템플릿에 데이터를 넣어 응답 HTML을 만든다.

강의는 EJS의 장점으로 익숙한 문법, 서버 사이드 렌더링, 적은 학습 부담을 설명한다. JavaScript를 이미 알고 있다면 템플릿 문법을 빠르게 읽을 수 있다.

SECTION 02

Express에 EJS 등록

 
app.set("view engine", "ejs");
app.set("views", "./views");

app.get("/notes", (req, res) => {
  res.render("notes", {
    title: "Current Notes",
    notes: ["공통 루틴 영역", "보안 점검", "배포 전 확인"],
  });
});

`app.set("view engine", "ejs")`는 Express가 `.ejs` 파일을 템플릿으로 렌더링하도록 설정한다. `views` 경로를 지정하면 Express는 그 폴더에서 템플릿 파일을 찾는다. 라우트에서는 `res.render("notes", data)`처럼 파일 이름과 데이터 객체를 넘긴다.

EJS 문법

 
<h1><%= title %></h1>

<% if (notes.length === 0) { %>
  <p>Empty Notes.</p>
<% } else { %>
  <ul>
    <% notes.forEach((note) => { %>
      <li><%= note %></li>
    <% }) %>
  </ul>
<% } %>
문법의미주의
<%= value %>값을 HTML 이스케이프하여 출력한다.사용자 입력 출력에 기본적으로 더 안전하다.
<%- value %>값을 이스케이프하지 않고 원문 HTML로 출력한다.XSS 위험이 커서 신뢰된 HTML에만 사용한다.
<% code %>조건문, 반복문 등 실행 코드를 넣는다.출력은 하지 않는다.
<%- include("file") %>다른 템플릿 조각을 포함한다.공통 헤더, 푸터, 메뉴에 자주 사용한다.
SECTION 04

출력과 보안

EJS는 서버에서 HTML을 만드는 도구이므로 사용자의 입력이 템플릿으로 들어갈 때 주의해야 한다. 사용자 입력은 템플릿 코드가 아니라 데이터로만 전달해야 한다. 출력할 때는 기본적으로 이스케이프되는 `<%= %>`를 사용하고, `<%- %>`는 반드시 신뢰 가능한 HTML에만 제한해야 한다.

다음 강의의 EJS RCE는 이 경계가 깨질 때 어떤 일이 생기는지 다룬다. 템플릿 파일, 렌더 옵션, 사용자 데이터의 경계를 분명히 구분하는 것이 핵심이다.
