학습 목표

  • 소켓이 애플리케이션과 전송 계층 사이의 통신 출입구라는 점을 설명한다.

소켓 프로그래밍의 목적

마지막 강의는 소켓을 이용해 클라이언트-서버 애플리케이션을 어떻게 만드는지 소개한다. 우리가 사용하는 대부분의 애플리케이션은 네트워크를 거치며, 그 내부에는 TCP 또는 UDP 기반 소켓 프로그래밍이 들어 있다. 이 강의에서는 UDP와 TCP를 기준으로 기본 구조를 본다.

소켓은 앞 강의에서 설명한 것처럼 애플리케이션과 전송 계층 사이의 출입구다. 애플리케이션은 소켓을 통해 데이터를 보내고 받고, 전송 계층은 TCP나 UDP 규칙에 맞춰 세그먼트 또는 데이터그램을 처리한다.

UDP 소켓 프로그래밍 흐름

UDP는 비연결형 프로토콜이다. 데이터를 보내기 전에 handshake로 연결을 설정하지 않는다. 송신자는 각 패킷에 목적지 IP와 port만 붙여 보내면 되고, 수신자는 받은 패킷에서 보낸 사람의 IP와 port를 추출한다. 다만 UDP는 신뢰성이 없기 때문에 전송된 데이터가 손실될 수 있다.

서버 소켓

UDP socket 생성 후 port bind

←
클라이언트

서버 IP와 port로 datagram 전송

→
응답

서버가 client address로 datagram 반환

서버와 클라이언트는 각각 소켓을 열고, 클라이언트는 서버 IP와 port로 데이터그램을 보낸다. 서버는 UDP 세그먼트를 읽고 필요한 처리를 한 뒤 응답을 보낸다. 클라이언트는 응답을 받고 작업을 마치면 소켓을 닫는다.

UDP client 예시

강의에서 설명한 Python UDP client 구조를 학습용으로 정리하면 다음과 같다. AF_INET은 IPv4를 의미하고, SOCK_DGRAM은 UDP socket을 의미한다. Client는 메시지를 입력받아 서버 주소로 보내고, 서버 응답을 기다린 뒤 출력하고 socket을 닫는다.

from socket import socket, AF_INET, SOCK_DGRAM

server_name = "127.0.0.1"
server_port = 12000

client_socket = socket(AF_INET, SOCK_DGRAM)
message = input("Input lowercase sentence: ")

client_socket.sendto(message.encode(), (server_name, server_port))
modified_message, server_address = client_socket.recvfrom(2048)

print(modified_message.decode())
client_socket.close()

UDP server 예시

UDP server는 UDP socket을 만들고 12000번 port에 bind한다. 이후 계속 반복문을 돌며 client에서 온 데이터를 recvfrom으로 받고, 처리한 결과를 client address로 다시 보낸다. UDP에서는 매 요청마다 sender address를 함께 확인하는 점이 중요하다.

from socket import socket, AF_INET, SOCK_DGRAM

server_port = 12000
server_socket = socket(AF_INET, SOCK_DGRAM)
server_socket.bind(("", server_port))

print("UDP server is ready")

while True:
    message, client_address = server_socket.recvfrom(2048)
    modified_message = message.decode().upper()
    server_socket.sendto(modified_message.encode(), client_address)

TCP 소켓 프로그래밍 흐름

TCP는 UDP와 달리 데이터를 주고받기 전에 연결이 수립되어야 한다. 서버에는 항상 열려 있는 welcome socket이 있고, 클라이언트는 그 socket에 연결하고 싶다고 요청한다. TCP connection이 이루어지면 서버는 welcome socket과 별개의 connection socket을 만들어 실제 데이터 통신에 사용한다.

이 구조 덕분에 서버는 여러 클라이언트와 통신할 수 있고, 각 클라이언트는 source port 등으로 구분된다. TCP는 신뢰성 있는 연결을 제공하며, 3-way handshake 이후 connection socket을 통해 데이터가 오간다.

요소역할
welcome socket서버가 클라이언트 연결 요청을 기다리는 소켓이다.
connect클라이언트가 서버와 TCP connection을 만들기 위해 호출한다.
listen서버가 연결 요청을 받을 준비를 한다.
accept연결 요청을 받아 connection socket을 만든다.
connection socket연결된 client와 실제 데이터를 주고받는 소켓이다.

TCP client 예시

TCP client는 UDP client와 달리 데이터를 보내기 전에 connect를 호출한다. 이 호출을 통해 TCP connection이 만들어지고, 이후 같은 socket으로 데이터를 보내고 응답을 받는다.

from socket import socket, AF_INET, SOCK_STREAM

server_name = "127.0.0.1"
server_port = 12000

client_socket = socket(AF_INET, SOCK_STREAM)
client_socket.connect((server_name, server_port))

sentence = input("Input lowercase sentence: ")
client_socket.send(sentence.encode())

modified_sentence = client_socket.recv(2048)
print(modified_sentence.decode())

client_socket.close()

TCP server 예시

TCP server는 socket을 만들고 bind한 뒤, listen으로 연결 요청을 기다린다. Client가 연결하면 accept가 connection socket을 반환하고, server는 그 socket으로 값을 받고 응답을 보낸 뒤 connection socket을 닫는다. Welcome socket은 계속 살아 있으므로 다음 client 연결을 다시 받을 수 있다.

from socket import socket, AF_INET, SOCK_STREAM

server_port = 12000
server_socket = socket(AF_INET, SOCK_STREAM)
server_socket.bind(("", server_port))
server_socket.listen(1)

print("TCP server is ready")

while True:
    connection_socket, client_address = server_socket.accept()
    sentence = connection_socket.recv(2048).decode()
    modified_sentence = sentence.upper()
    connection_socket.send(modified_sentence.encode())
    connection_socket.close()

UDP와 TCP 구현 차이

구분UDPTCP
연결 설정없다.connect와 3-way handshake가 있다.
전송 함수 감각sendto/recvfrom으로 주소를 함께 다룬다.연결 후 send/recv로 통신한다.
서버 구조bind 후 반복적으로 datagram을 읽는다.listen 후 accept로 connection socket을 만든다.
신뢰성기본 제공하지 않는다.신뢰성 있는 전송을 제공한다.
종료클라이언트가 할 일을 마치면 socket을 닫는다.통신이 끝난 connection socket을 닫고 welcome socket은 계속 유지할 수 있다.
