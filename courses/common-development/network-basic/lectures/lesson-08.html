SECTION 01

소켓 프로그래밍의 목적

마지막 강의는 socket programming으로 client/server 애플리케이션을 어떻게 만드는지 소개한다. 우리가 사용하는 대부분의 애플리케이션은 네트워크를 거치며, 내부에서는 TCP 또는 UDP 기반 소켓을 통해 데이터를 주고받는다.

소켓은 호스트의 application layer와 transport layer 사이의 인터페이스다. 애플리케이션은 소켓에 데이터를 넣고, transport layer는 TCP나 UDP 규칙에 맞춰 segment 또는 datagram을 처리한다. 강의에서는 소켓 유형을 크게 UDP socket과 TCP socket으로 나누어 본다.

이 강의의 목표는 완성된 대규모 서버 프로그램을 만드는 것이 아니라, 네트워크 애플리케이션의 최소 구조를 이해하는 것이다. 즉 server와 client가 각각 socket을 만들고, 어떤 함수로 데이터를 보내고 받으며, TCP에서는 왜 연결 설정이 먼저 필요한지를 파악하는 데 집중한다.

SECTION 02

UDP 소켓 프로그래밍 흐름

UDP는 비연결형 프로토콜이다. 데이터를 보내기 전에 TCP처럼 handshake로 연결을 설정하지 않는다. Sender는 각 packet에 destination IP와 port를 붙여 보내고, receiver는 수신된 packet에서 보낸 사람의 IP와 port를 추출한다.

그림에서 server와 client는 각각 socket을 연다. 이때 UDP socket은 socket(AF_INET, SOCK_DGRAM)으로 만든다. AF_INET은 IPv4를 의미하고, SOCK_DGRAM은 UDP datagram socket을 의미한다.

서버 소켓

UDP socket 생성 후 port bind

←
클라이언트

서버 IP와 port로 datagram 전송

→
응답

서버가 client address로 datagram 반환

Client는 message를 datagram으로 만들어 server IP와 port로 보낸다. Server는 UDP segment를 읽고, 처리 결과를 다시 client address와 port로 보낸다. Client는 응답을 받은 뒤 할 일을 마치면 socket을 닫는다. 흐름은 간단하지만 UDP는 신뢰성을 제공하지 않기 때문에 전송 데이터가 손실될 수 있다는 점을 함께 기억해야 한다.

SECTION 03

UDP client 예시

강의 화면의 UDP client는 Python socket 모듈을 사용한다. serverName에는 server host name 또는 IP가 들어가고, serverPort에는 12000이 들어간다. 그 다음 clientSocket = socket(AF_INET, SOCK_DGRAM)으로 IPv4 기반 UDP socket을 생성한다.

from socket import * # 해당 모듈을 통해 소켓 생성

serverName = 'hostname'
serverPort = 12000

clientSocket = socket(AF_INET, SOCK_DGRAM) # client socket 생성, IPv4, UDP

message = Input('Input lowercase sentence:')

clientSocket.sendto(message.encode(), (serverName, serverPort))

# 패킷 데이터, 패킷 출발지 주소 할당
modifiedMessage, serverAddress = clientSocket.recvfrom(2048)

print(modifiedMessage.decode())

# 소켓 닫기
clientSocket.close()

sendto는 UDP에서 중요한 함수다. TCP처럼 먼저 연결된 상대가 있는 것이 아니므로, 보낼 message와 함께 목적지인 (serverName, serverPort)를 매번 넘겨 준다. recvfrom(2048)은 server의 응답 data와 그 응답이 온 server address를 함께 반환한다. 마지막으로 client는 clientSocket.close()로 socket을 닫는다.

SECTION 04

UDP server 예시

UDP server도 마찬가지로 socket(AF_INET, SOCK_DGRAM)으로 UDP socket을 만든다. Server는 client가 찾아올 수 있도록 특정 port를 socket에 묶어야 하므로 serverSocket.bind(('', serverPort))를 호출한다. 강의 예시는 12000번 port를 사용한다.

from socket import *

serverPort = 12000
serverSocket = socket(AF_INET, SOCK_DGRAM)

# 12000 포트 번호를 소켓 할당
serverSocket.bind(('', serverPort))

print("The server is ready to receive")

while True:
    # 데이터, 패킷 출발지 주소 저장
    message, clientAddress = serverSocket.recvfrom(2048)

    modifiedMessage = message.decode().upper()

    serverSocket.sendto(modifiedMessage.encode(), clientAddress)

Server는 while True 반복문 안에서 계속 client 요청을 기다린다. recvfrom(2048)은 client가 보낸 message와 client address를 함께 반환한다. Server는 message를 decode한 뒤 upper()로 대문자로 바꾸고, sendto를 이용해 같은 client address로 다시 보낸다. UDP server는 연결을 유지하지 않으므로 매 요청에서 sender address를 함께 다루는 구조가 된다.

SECTION 05

TCP 소켓 프로그래밍 흐름

TCP는 UDP와 달리 데이터를 주고받기 전에 연결이 수립되어야 한다. Server에는 client의 연결 요청을 기다리는 welcome socket이 열려 있고, client는 server IP와 port로 “연결하고 싶다”고 요청한다. 이 과정에서 TCP 3-way handshake가 수행된다.

연결이 수립되면 server는 welcome socket과 별개의 connection socket을 생성한다. 실제 data 송수신은 이 connection socket을 통해 이루어진다. Welcome socket은 계속 열려 있으므로 server는 다음 client의 연결 요청도 받을 수 있다.

이 구조 덕분에 server는 여러 client와 통신할 수 있고, 각 client는 source port 등으로 구분된다. TCP는 client-server 사이에 신뢰성 있는 연결을 제공하며, 연결 설정 이후에는 byte stream 형태로 데이터를 주고받는다.

요소역할
welcome socket서버가 클라이언트 연결 요청을 기다리는 소켓이다.
connect클라이언트가 서버와 TCP connection을 만들기 위해 호출한다.
listen서버가 연결 요청을 받을 준비를 한다.
accept연결 요청을 받아 connection socket을 만든다.
connection socket연결된 client와 실제 데이터를 주고받는 소켓이다.
SECTION 06

TCP client 예시

TCP client는 UDP client와 마찬가지로 server name과 server port를 지정하지만, socket type이 다르다. socket(AF_INET, SOCK_STREAM)은 IPv4 기반 TCP socket을 의미한다. UDP의 SOCK_DGRAM과 달리 TCP는 SOCK_STREAM을 사용한다.

from socket import *

serverName = 'servername'
serverPort = 12000

# IPv4, TCP 소켓 명시
clientSocket = socket(AF_INET, SOCK_STREAM)

# 데이터를 보내기 전 TCP connection이 먼저 이루어져야 한다. (3-way handshake)
clientSocket.connect((serverName, serverPort))

sentence = raw_input('Input lowercase sentence:')
clientSocket.send(sentence.encode())

modifiedSentence = clientSocket.recv(1024)
print('From Server: ', modifiedSentence.decode())

clientSocket.close() # 연결 종료

가장 중요한 차이는 clientSocket.connect((serverName, serverPort))다. TCP는 데이터를 보내기 전에 연결이 먼저 이루어져야 하므로, 이 connect 호출에서 TCP connection setup, 즉 3-way handshake가 수행된다. 그 다음에는 send로 데이터를 보내고, recv(1024)로 server response를 받는다. 통신이 끝나면 client socket을 닫는다.

SECTION 07

TCP server 예시

TCP server는 먼저 serverSocket = socket(AF_INET, SOCK_STREAM)으로 TCP socket을 만든다. 그 다음 bind로 12000번 port에 server socket을 묶고, listen(1)으로 client가 문을 두드리기를 기다린다.

from socket import *

serverPort = 12000
serverSocket = socket(AF_INET, SOCK_STREAM) # TCP 소켓 생성

serverSocket.bind(('', serverPort))

serverSocket.listen(1) # 클라이언트가 문을 두드리기를 기다린다.
print('The server is ready to receive')

while True:
    # client가 TCP 연결 요청을 하면, accept()로 연결 소켓 생성
    connectionSocket, addr = serverSocket.accept()
    sentence = connectionSocket.recv(1024).decode()
    capitalizedSentence = sentence.upper()
    connectionSocket.send(capitalizedSentence.encode())

    connectionSocket.close() # 응답을 보내고 연결 소켓을 닫는다.

Client가 TCP 연결 요청을 보내면 serverSocket.accept()가 호출되고, 이때 connectionSocket이 만들어진다. Server는 이 connection socket에서 recv(1024)로 문장을 받고, upper()로 대문자로 바꾼 뒤 send로 돌려준다. 마지막에 닫는 것은 welcome socket이 아니라 해당 client와 통신하던 connection socket이다.

Q. 소켓 프로그래밍 그리고 TCP 소켓 프로그래밍을 어떻게 진행할 수 있는지를 소개를 드리려고 합니다. 어떻게 소켓을 이용해서 클라이언트 서버 애플리케이션을 만드는가 저희가 쓰는 대부분의 애플리케이션들은 네트워크를 거치고 이런 소켓 프로그래밍이 되어 있겠죠. 그래서 흔히 UDP와 TCP 기반으로 소켓 프로그래밍을 할 수 있고요. 이거를 더 커스텀해서 할 수도 있습니다. 그래서 제가 지금 다룰 것은 UDP와 TCP이고요. 먼저 UDP 소켓 프로그래밍입니다. UDP는 비연결성이 특징이라고 했는데요. 그래서 연결 설정을 하는 핸드쉐이크 과정도 없고요. 단지 샌더는 각 패킷에 목적지 IP와 포트만 첨부를 하면 값을 보낼 수 있게 됩니다. 그러면 그 리시버는 수신된 패킷에서 보내온 사람의 IP와 포트가 뭔지를 추출하는 방식으로 진행이 되는 거죠. 그래서 UDP는 정선된 데이터가 손실이 될 수 있어요. 그러니까 신뢰성이 없다는 뜻이죠. 지금 보이시는 그림은 UDP 소켓 프로그래밍에 대한 계략적인 요약들인데요. 지금 보시면 서버와 클라이언트는 각각 소켓을 열어줍니다.

이 소켓을 열어준다는 뜻은 이제 어플리케이션과 트랜스포스 계층과 통신을 하기 위한 그런 출입구라고 할 수 있다고 했죠. 그래서 지금 보시면 각각 소켓을 연 다음에 클라이언트는 데이터그램을 서버 IP와 포트로 값을 보내게 됩니다. 그럼 서버는 UDP 세그먼트를 읽어드려서 그거에 맞는 응답을 주게 되는 거죠. 그러면 클라이언트는 그거를 다시 받아서 할 일을 다 마치면 클로즈를 하게 되는 비교적 간단한 방식입니다. 그래서 지금 보이시는 코드는 UDP 클라이언트를 파이썬 소켓으로 구현한 코드인데요. 지금 보시면 파이썬의 소켓이라는 모듈을 통해서 소켓을 생성할 수 있습니다. 그래서 이 클라이언트 소켓이라는 변수명으로 소켓을 생성하죠. 클라이언트 소켓을. 여기서 보이시는 AF-iNET은 IP 버전 4를 얘기하고요. 그리고 속 Sock-dj-ram은 UDP를 나타냅니다. 그리고 이 소켓, 열어진 소켓으로 값을 보내게 되죠. 메시지를 입력받아서 서버로 보내게 됩니다. 그러면 그 서버로부터의 응답을 기다린 다음에 값을 출력하고 할 일을 다 마치고 소켓을 닫는 구조로 간단하게 지금 이루어져 있죠.

하지만 여기서 신뢰성은 없다는 얘기를 다시 한번 얘기를 드리고 다음은 UDP 서버는 어떻게 구현하는지 설명을 드리겠습니다. UDP도 똑같이 소켓을 UDP로 열어줍니다.
