보안 통신과 세 가지 보안 목표

보안 통신(secure communications)은 정보 시스템 보안을 위한 기본 목표가 달성된 통신을 말한다. 강의에서는 보안의 세 가지 기본 목표로 기밀성(confidentiality), 무결성(integrity), 가용성(availability)을 든다.

보안 목표의미
기밀성인가받은 사용자만 메시지를 읽을 수 있도록 보장한다.
무결성데이터가 수정되거나 변조되지 않았음을 확인한다.
가용성필요한 사용자가 필요한 때 정보 시스템을 사용할 수 있어야 한다.

보안 목표를 달성하기 위해 사용되는 기술들을 security mechanism이라고 한다. 강의에서는 encipherment, data integrity, digital signature, authentication exchange, traffic padding, routing control, notarization, access control을 예로 든다.

보안 서비스

공개 채널, 예를 들어 인터넷에서 안전하게 데이터를 주고받기 위해 제공해야 하는 기능을 security service라고 볼 수 있다. 강의에서는 다섯 가지를 중심으로 설명한다.

서비스강의에서의 설명
데이터 기밀성authorized된 사용자만 메시지를 읽을 수 있게 한다.
데이터 무결성데이터가 수정되지 않았음을 확인한다. 악의적 변조도 수정에 포함된다.
데이터 출처 인증메시지가 실제 출처에서 왔고 내용이 수정되지 않았음을 확인한다.
부인방지송신자가 메시지를 보낸 사실을 나중에 부정할 수 없게 한다.
접근 통제승인된 사용자만 데이터 또는 단말기에 접근하도록 한다.

강의는 영어 용어에 익숙해지는 것도 중요하다고 강조한다. 보안과 암호 기술에서는 confidentiality, integrity, authentication, non-repudiation, access control 같은 영어 단어가 계속 등장하기 때문이다.

무결성, 인증, 부인방지의 차이

무결성은 메시지가 중간에서 바뀌지 않았는지를 확인하는 것이다. 인증은 그 메시지가 누구로부터 왔는지를 확인하는 것이다. A가 B에게 메시지를 보냈다면, B가 받은 메시지가 정말 A로부터 왔다는 것을 확인하는 것이 인증이다.

부인방지는 여기서 한 단계 더 나아간다. A가 B에게 메시지를 보낸 뒤 나중에 “나는 그런 메시지를 보낸 적이 없다”고 부정하지 못하게 하는 것이다. 인증은 수신자가 출처를 확인하는 것이고, 부인방지는 송신자가 행위 자체를 부정하지 못하게 하는 보안 서비스다.

암호학 용어

암호 기술은 통신 당사자 간 메시지 전달 과정을 보호하고, 컴퓨터에 저장된 메시지를 보호하기 위한 수학적 기술이다. 강의는 비슷하게 보이는 여러 영어 용어를 구분한다.

용어의미
Cryptology비밀 코드, 즉 암호화된 코드를 만들거나 해독하는 학문 전체다.
Cryptography평문을 암호문으로 만드는 암호 기술 자체를 가리킨다.
Cryptanalysis암호화된 코드를 분석하고 해독하는 기술이다.
Crypto암호를 의미하는 포괄적 표현이다.

이 강의에서는 주로 cryptography라는 단어, 즉 암호 기술을 중심으로 다룬다.

Kerckhoffs의 원리

강의는 암호 기술을 공부하기 전에 반드시 기억해야 할 원리로 Kerckhoffs의 원리를 소개한다. Kerckhoffs는 프랑스의 언어학자이자 암호학자로, 1880년대에 군사 암호 기술과 관련된 글에서 실용적인 암호 설계 원칙을 제안했다.

  • 암호 시스템은 수학적으로 완전히 불가능하지 않더라도 실질적인 시간 안에는 해독할 수 없어야 한다.
  • 암호 시스템은 비밀성을 요구하지 않아야 하며, 적의 손에 넘어가도 문제가 되지 않아야 한다.
  • 비밀키는 전달하고 기억할 수 있어야 하며, 필요하면 변경하거나 수정할 수 있어야 한다.
  • 전신 통신에서도 사용할 수 있어야 한다. 현대적으로는 디지털 통신과 비트열로 사용할 수 있어야 한다는 뜻으로 이해할 수 있다.
  • 암호 기술은 휴대 가능해야 하며 여러 사람이 복잡하게 조작해야만 하는 방식이면 안 된다.
  • 암호 시스템은 사용하기 쉬워야 하고 사용자에게 복잡한 규칙을 과도하게 요구해서는 안 된다.

멘토가 특히 강조하는 것은 두 번째 원칙이다. 암호 알고리즘 자체는 언젠가 공개될 수 있고, 현대 암호에서는 실제로 알고리즘과 구현 코드가 공개되는 경우가 많다. 안전성은 알고리즘을 숨기는 데서 나오지 않고, 공개된 알고리즘 위에서 비밀키를 안전하게 보호하는 데서 나온다.

알고리즘과 비밀키

암호 기술은 알고리즘과 비밀키가 함께 사용된다. 로그인할 때 아이디와 패스워드를 사용하는 것처럼, 인터넷 뱅킹의 인증서 비밀번호를 잘 보호해야 하는 것처럼, 핵심은 비밀키 또는 비밀번호의 보호다.

알고리즘은 누구나 알 수 있다는 전제 위에서 수학적으로 안전해야 한다. 반면 비밀키는 안전하게 보관되어야 한다. 하지만 비밀키가 너무 기억하기 어렵거나 사용하기 불편해서 포스트잇에 적어 책상에 붙여 놓는다면 그것은 보안 시스템의 실패가 된다. 강의는 안전성과 사용성이 함께 고려되어야 함을 강조한다.

보안 강도와 완전 탐색

암호 기술에서 안전성 수준은 보안 강도(security strength)로 표현된다. 보안 강도는 암호 시스템, 암호키, 해시 함수의 취약성을 찾아내는 데 필요한 작업량을 수치화한 것이다. 예를 들어 112비트 보안 강도는 대략 2의 112승 수준의 계산을 해야 취약성이나 키를 찾아낼 수 있다는 의미로 설명된다.

완전 탐색(exhaustive search)은 가능한 모든 키를 하나씩 시도하는 방식이다. n비트 비밀키가 사용되었다면 가능한 키는 2의 n승 개이고, 평균적으로는 2의 n-1승 정도의 시도가 필요하다고 설명한다. 현대 암호는 보통 112비트 이상의 키 길이를 요구한다.

암호 알고리즘과 키 길이 선택

암호 알고리즘과 키 길이는 서로 연결되어 있다. 어떤 알고리즘을 사용할지, 얼마만큼의 키 길이를 사용할지는 알고리즘의 안전성 유지 기간과 보안 강도별 권장 키 길이를 비교해 결정해야 한다.

강의에서는 2018년 한국인터넷진흥원(KISA)의 암호 알고리즘 및 키 길이 이용 안내서를 예로 들며, 2030년까지 보호해야 하는 비밀은 112비트 보안 강도가 기준이 될 수 있고, 2030년 이후에는 128비트 이상이 요구된다고 설명한다. 같은 보안 강도라도 대칭키와 해시 함수는 비슷한 비트 수를 쓰지만, 공개키 암호 기술은 2048비트 또는 3072비트처럼 훨씬 긴 키가 필요하다는 점도 강조한다.

2030년 기준이 등장하는 이유로 강의는 양자 컴퓨터의 등장을 언급한다. 학계에서는 2026년 또는 2027년 정도가 되면 현재 공개키 암호 기술이 무력화될 수 있다는 예측이 있다고 설명하고, 대칭키 암호 기술도 더 긴 키를 사용하는 방향을 고려해야 한다고 말한다.

암호학자의 도구 상자

강의는 Bruce Schneier의 책 Secrets and Lies, 국내 번역 제목으로는 보안과 거짓말에 나오는 “암호학자의 도구 상자”를 언급한다. 여기에는 메시지 인증 코드, 일방향 해시 함수, 디지털 서명, 공개키 암호, 대칭 암호, 의사 난수 생성기 같은 도구가 포함된다.

이 도구들은 이후 강의에서 하나씩 다시 등장한다. 메시지 인증 코드와 일방향 함수는 해시와 연결되고, 디지털 서명은 해시와 공개키 암호를 결합하며, 대칭키와 공개키 암호는 기밀성·인증·부인방지를 달성하는 핵심 도구가 된다.

위협과 암호 기술의 대응

보안 위협과 암호 기술은 서로 연결된다. 도청은 기밀성을 위협하고, 메시지 변경은 무결성을 위협하고, 위장은 인증을 위협하고, 부인은 부인방지를 위협한다. 여기에 대응하기 위해 대칭키 암호, 공개키 암호, 일방향 해시 함수, 메시지 인증 코드, 디지털 서명이 사용된다.

암호 기술의 큰 분류

암호 기술은 암호화와 복호화의 방향성에 따라 일방향과 양방향으로 구분한다. 일방향 암호 기술은 암호화만 가능한 기술이고, 대표적인 예가 해시 함수다. 해시 함수가 MDC로 사용될 때는 무결성을 달성하고, MAC으로 사용될 때는 무결성과 인증을 함께 달성할 수 있다. 예로는 MD5, SHA-2, SHA-3, HMAC이 언급된다.

양방향 암호 기술은 암호화와 복호화를 모두 제공한다. 키의 유형에 따라 대칭키 암호 기술과 비대칭키 암호 기술로 나뉜다. 대칭키는 암호화와 복호화에 같은 키를 사용하며, stream 암호와 block 암호로 나눌 수 있다. 비대칭키는 암호화 키와 복호화 키가 서로 다르며 공개키 암호 기술이라고도 한다. 공개키 암호 기술은 기밀성, 인증, 부인방지까지 연결된다.
