SECTION 01

리눅스 파일시스템을 먼저 봐야 하는 이유

5강은 박소연 멘토가 진행하는 리눅스 기초 명령어 수업이다. 오늘의 핵심은 파일과 디렉토리를 다루는 법이다. 명령어 자체를 외우기 전에, 리눅스 안에서 파일과 디렉토리가 어떤 구조로 놓이는지 알아야 한다.

리눅스 파일시스템은 FHS, Filesystem Hierarchy Standard라는 표준 구조를 따른다. 최상단에는 루트 디렉토리 /가 있고, 그 아래에 /bin, /boot, /dev, /etc, /home, /lib, /media, /mnt, /proc, /root, /sbin, /tmp, /usr, /var 같은 디렉토리가 놓인다. 처음에는 복잡해 보이지만 모든 리눅스의 기본 표준이므로 자주 보는 위치는 암기하는 것이 좋다.

디렉토리강의에서 강조한 의미
/모든 디렉토리의 최상단인 루트 디렉토리다.
/bin일반 사용자가 실행하는 기본 명령어 바이너리가 놓인다.
/boot부트로더, 커널, initrd처럼 부팅에 필요한 파일이 있다.
/dev디스크, 터미널 같은 시스템 장치를 파일처럼 표현하는 장치 파일이 있다.
/etc각종 시스템 설정 파일이 모이는 매우 중요한 위치다.
/homeuser1, user2 같은 일반 사용자 홈 디렉토리가 놓인다.
/root최상위 관리자 root 사용자의 홈 디렉토리다.
/sbinsuper user, 즉 관리자용 시스템 명령어가 놓이는 위치다.
/tmp임시 파일 저장소다.
/usr일반 사용자를 위한 공통 프로그램과 라이브러리가 많이 놓인다.
/var시스템 운용 중 계속 변하는 임시 데이터, 로그, 캐시 등이 저장된다.

강사는 특히 /etc와 /var를 반드시 알고 있어야 한다고 강조한다. 서버를 운영하고 문제를 분석할 때 가장 많이 들어가게 되는 곳이 설정 파일이 모인 /etc와 로그가 쌓이는 /var/log이기 때문이다.

SECTION 02

/boot, /home, /root, /etc, /var/log

강의는 표준 구조를 설명한 뒤 실제 디렉토리를 몇 군데 들어가 본다. 먼저 /boot는 리눅스가 부팅될 때 필요한 파일이 있는 곳이다. ls -al /boot로 보면 grub, initrd, memtest, vmlinuz 같은 파일이 보인다.

vmlinuz로 보이는 파일들이 Linux 커널이다. 강사는 Ubuntu 16.04 기준 커널 파일 하나가 약 8MB 정도에 불과하다고 설명한다. 운영체제의 핵심 기능, 즉 프로세스 관리, 네트워크 관리, 파일시스템 관리, 하드웨어 관리 같은 핵심이 이 작은 커널에 담겨 있다. 다만 사용자 프로그램과 부팅에 필요한 나머지 도구는 initrd 같은 별도 파일에 포함된다.

/home에는 일반 사용자 홈 디렉토리가 있다. 수업에서는 user1만 있으면 감이 덜 오므로 user2도 만들어 여러 사용자가 한 시스템 안에 공존할 수 있음을 보여 준다. 반대로 /root는 root 사용자의 홈 디렉토리라 일반 사용자가 그냥 보려고 하면 권한이 없어 접근이 제한된다. sudo ls -l /root처럼 관리자 권한을 빌리면 볼 수 있는데, 이 sudo와 권한 개념은 뒤 강의에서 자세히 배운다.

/etc는 설정 파일 디렉토리다. 화면에는 “설명 파일”이라고 적힌 부분이 있지만 강사가 바로 “설정 파일”이 맞다고 고쳐 설명한다. 이곳에는 네트워크 설정, AppArmor 같은 보안 기능 설정, APT 패키지 관리자 설정, 웹 서버 설정, 사용자 계정 정보가 들어 있는 passwd 파일 등 시스템을 움직이는 핵심 설정이 모인다. 잘못 바꾸면 시스템에 치명적인 문제가 생길 수 있으므로 의미를 알고 조심해서 다뤄야 한다.

/var는 시스템 운용 중 변하는 파일이 쌓이는 위치다. 웹 서버를 설치하면 웹 콘텐츠가 들어갈 수 있고, 캐시, crash 파일, lock 파일, 임시 데이터도 생긴다. 그중 가장 중요한 곳은 /var/log다. 프로그램을 설치했는데 잘 되었는지 모르거나, 오류가 났거나, 어떤 일이 있었는지 분석해야 할 때 로그 파일을 본다. 서버 운영자는 /var/log를 자주 확인하게 된다.

SECTION 03

터미널 프롬프트와 clear

파일시스템을 훑은 뒤 기본 명령어로 들어간다. 앞으로 실습은 별도 설명이 없어도 터미널을 띄운 뒤 그 안에서 명령어를 입력한다고 보면 된다. 가장 먼저 보는 명령은 clear다. 터미널에 명령어 출력이 많이 쌓여 지저분할 때 clear를 입력하면 화면이 깨끗해진다. 데이터가 삭제되는 것이 아니라 화면 출력만 지워지는 것이다.

clear

터미널 맨 끝의 $ 표시는 프롬프트(prompt)다. 운영체제가 “명령어를 기다리고 있다”는 뜻으로 보여 주는 표시다. 프롬프트 앞에 보이는 ~는 틸드라고 부르며, 현재 사용자의 홈 디렉토리를 의미한다. 이 수업의 user1 기준으로는 /home/user1을 짧게 ~로 보여 주는 것이다.

Windows에도 파일 탐색기뿐 아니라 명령 프롬프트가 있고, 숨김 파일 표시 옵션이 있다. Linux도 마찬가지로 GUI 없이 터미널 명령어만으로 파일과 디렉토리를 만들고, 보고, 옮기고, 지울 수 있다. 이 강의는 그 기본 명령어들을 하나씩 익히는 시간이다.

SECTION 04

ls: 파일 목록 보기와 숨김 파일

ls는 list의 약자다. 오래된 UNIX 계열 명령어는 역사적으로 짧은 약어가 많다. 예전에는 글자 하나하나도 메모리와 입력 비용이었고, 명령어를 길게 치는 것이 비효율적이었다. 그래서 오늘날에도 list가 아니라 ls처럼 짧은 형태가 남아 있다. 강사는 약자를 풀어 외우면 나중에 기억하기 쉽다고 설명한다.

ls
ls -l
ls -a
ls -al
ls -la
ls -a -l
ls *.txt
명령의미
ls현재 디렉토리의 파일과 디렉토리 목록을 보여 준다.
ls -llong list. 권한, 소유자, 크기, 시간 같은 상세 정보를 함께 보여 준다.
ls -aall. 점으로 시작하는 숨김 파일까지 보여 준다.
ls -al, ls -la, ls -a -l숨김 파일까지 긴 형식으로 보여 준다. 옵션 순서는 바뀌어도 된다.
ls *.txt확장자가 .txt인 파일만 패턴으로 골라 보여 준다.

ls -al 결과의 첫 줄에는 .과 ..이 보인다. .은 현재 디렉토리, ..은 부모 디렉토리다. 그 외에 .bashrc처럼 점으로 시작하는 파일은 숨김 파일이다.

SECTION 05

touch와 cat: 파일 만들기와 내용 보기

touch는 원래 “파일을 툭 건드린다”는 의미에 가깝다. 정확한 기본 기능은 파일의 수정 시간을 현재 시간으로 바꾸는 것이다. 하지만 존재하지 않는 파일을 touch하면 해당 이름의 0바이트 파일이 만들어진다. 그래서 실무에서도 빈 파일을 빠르게 만들 때 자주 사용한다.

touch hello.txt
touch test1 test2 test3
touch .hello.txt
ls -l
ls -al
rm .hello.txt

touch .hello.txt는 기존 hello.txt를 숨기는 명령이 아니다. hello.txt와 .hello.txt는 서로 다른 파일이고, .hello.txt는 파일명이 점으로 시작하기 때문에 기본 ls -l에서는 보이지 않을 뿐이다. ls -al로 보면 숨김 파일까지 확인할 수 있다.

cat은 concatenate에서 온 이름이다. 연결한다는 뜻인데, 여기서는 파일이라는 입력(input)을 화면의 표준 출력(standard output)으로 연결해 내용을 보여 준다고 이해하면 된다. cat /etc/passwd는 사용자 계정 정보가 들어 있는 파일을 보여 주고, cat /var/log/syslog는 시스템 로그를 바로 출력한다.

cat hello.txt
cat /etc/passwd
cat /var/log/syslog
cat -e /etc/passwd
cat -n /etc/passwd

cat -e는 줄 끝에 $를 붙여 공백이나 숨은 문자를 확인하는 데 쓸 수 있고, cat -n은 줄 번호를 붙여 보여 준다. 다만 아주 긴 로그 파일을 무작정 cat으로 보면 화면이 한 번에 지나가 버리므로, 뒤에서 소개하는 more나 less가 더 적합할 때가 많다.

SECTION 06

more, less, rm: 긴 파일 보기와 삭제

more와 less는 긴 파일을 한 화면씩 넘겨 보게 해 주는 명령이다. cat은 파일 전체를 한 번에 출력하지만, 로그처럼 긴 파일은 한 번에 지나가 버려 읽기 어렵다. more는 아래로 내려가며 보고, less는 위아래 이동이 더 자유롭고 파일 전체를 메모리에 올리지 않아 큰 파일을 볼 때 속도도 유리하다.

more hello.txt
more /etc/passwd
more /var/log/syslog
less hello.txt
less /etc/passwd
less /var/log/syslog

rm은 remove의 약자로 파일을 삭제한다. 강사는 rm hello.txt, rm test1 test2 test3처럼 여러 파일을 삭제하는 예를 설명한다. 하지만 시스템 파일은 일반 사용자가 삭제할 수 없다. rm /etc/passwd처럼 중요한 파일을 지우려 하면 권한 문제로 실패한다. 이는 리눅스에 사용자와 권한 체계가 있고, 시스템 파일이 보호된다는 사실을 미리 보여 주는 예다.

rm hello.txt
rm test1 test2 test3
rm /etc/passwd
SECTION 07

mkdir, rmdir, cd: 디렉토리 만들기와 이동

mkdir은 make directory의 약자이고, rmdir은 remove directory의 약자다. mkdir dir1은 dir1을 만들고, mkdir dir2 dir3은 여러 디렉토리를 한 번에 만든다. mkdir dir1/sub1처럼 하위 디렉토리를 만들 수 있고, 부모 디렉토리까지 함께 만들려면 -p 옵션을 사용한다.

mkdir dir1
mkdir dir2 dir3
mkdir dir1/sub1
mkdir -p dir2/sub1
rmdir dir1
rmdir dir2 dir3
rmdir -p dir2
rm -r dir1

rmdir은 빈 디렉토리만 삭제할 수 있다. 안에 파일이나 하위 디렉토리가 있으면 실패한다. 반대로 rm -r은 recursive, 즉 재귀적으로 안쪽 파일과 디렉토리까지 지우므로 매우 위험하다. 실수로 중요한 위치에서 실행하면 큰 사고가 날 수 있어 반드시 주의해야 한다.

cd는 change directory의 약자다. cd dir1로 하위 디렉토리에 들어가고, cd ..으로 부모 디렉토리로 올라간다. cd ../..은 부모의 부모로 이동하고, cd ~ 또는 인자 없이 cd만 입력하면 홈 디렉토리로 돌아간다. cd -는 직전에 있던 디렉토리로 돌아간다. cd -를 반복하면 두 위치를 왔다 갔다 할 수 있다.

cd dir1
cd ..
cd dir1/sub1
cd .
cd ../..
cd ~
cd
cd -
cd /etc
cd /var/log

강사는 cd /etc, cd /var/log, cd로 홈에 돌아오기처럼 여러 경로를 직접 이동해 보라고 한다. 터미널에서 원하는 폴더로 자유롭게 이동할 수 있어야 이후 모든 CLI 실습이 가능하다.

SECTION 08

cp와 mv: 복사, 이동, 이름 변경

cp는 copy, mv는 move의 약자다. cp hello.txt hello2.txt는 파일을 복제하고, cp test1 dir1처럼 목적지가 존재하는 디렉토리라면 그 안으로 파일을 복사한다. 목적지가 디렉토리가 아니라 없는 이름이면, 그 이름의 새 파일로 복사될 수 있다. 이 차이를 알고 써야 한다.

touch hello.txt test1 test2
mkdir dir1
cp hello.txt hello2.txt
cp test1 dir1
cp test2 dir1
cp -r dir1 dir2
mv hello.txt hello2.txt
mv test1 dir1
mv test2 dir1
rm -r dir1 dir2
rm hello2.txt

디렉토리를 복사하려면 cp -r처럼 recursive 옵션이 필요하다. 강의 화면의 cleanup 명령은 실습 중 만든 dir1, dir2, hello2.txt를 정리하기 위한 예다. 복사와 이동 명령은 파일명을 바꾸는 데도 쓰이므로, “목적지가 파일인지 디렉토리인지”를 항상 확인해야 한다.

SECTION 09

ln, 하드링크, 심볼릭 링크, inode

ln은 link를 만드는 명령이다. Windows의 바로가기처럼 특정 파일을 다른 위치에서 빠르게 가리키는 기능을 떠올리면 이해하기 쉽다. 강의에서는 초급 단계에서 하드링크는 개념만 알고, 실무에서 자주 쓰는 심볼릭 링크를 꼭 알아야 한다고 설명한다.

touch hello.txt
ln hello.txt hellolink
ln -s hello.txt hellosymlink
ls -ali

하드링크는 두 파일명이 같은 실제 파일 데이터를 가리키는 구조다. 복사와 달리 데이터를 두 배로 차지하지 않고, 한쪽 이름으로 내용을 바꾸면 같은 데이터를 바라보는 다른 이름에서도 바뀐 내용이 보인다. 심볼릭 링크는 ln -s로 만들며, 실제 파일 데이터가 아니라 다른 파일 경로를 가리킨다. 그래서 Windows 바로가기와 더 비슷하다.

이 설명은 inode 개념과 이어진다. 우리가 파일명 자체가 파일이라고 생각하기 쉽지만, 실제 데이터는 inode를 거쳐 접근된다. inode에는 소유자, 권한(mode), 파일 크기, 생성·변경 시간(timestamp), 직접/간접 블록 위치 같은 메타데이터가 들어 있다. ls -ali를 보면 하드링크가 같은 inode 번호를 공유하고, 심볼릭 링크는 별도 inode를 갖고 대상 경로를 가리키는 모습을 확인할 수 있다.

SECTION 10

file과 man: 속성 확인과 도움말

file 명령은 대상 파일이 무엇인지 판별한다. 빈 텍스트 파일인지, /etc/passwd 같은 ASCII text인지, 디렉토리인지, 실행 파일인지, 심볼릭 링크인지 확인할 수 있다. 파일 확장자만 믿지 않고 실제 내용을 보고 유형을 추정한다는 점이 중요하다.

file hello.txt
file /etc/passwd
file dir1
file /usr/bin/file
file hellosymlink

명령어와 옵션을 전부 외울 수는 없다. 예전에는 구글이나 인터넷이 없었으므로, 시스템 안에서 바로 도움말을 볼 수 있는 매뉴얼이 필요했다. 그 도구가 man이다.

man ls
man file
man man

man ls는 ls 명령의 설명과 옵션을 보여 주고, man file은 file 명령의 설명을 보여 준다. 심지어 man man처럼 매뉴얼 자체의 매뉴얼도 볼 수 있다. 강사는 모르는 옵션을 그때그때 검색만 하기보다, 리눅스 안의 매뉴얼을 읽는 습관을 들이라고 설명한다.

SECTION 11

Vim과 Nano: GUI 없는 서버에서 편집하기

강의 마지막 이론 파트는 편집기 숙제다. 서버에는 GUI가 없을 수 있으므로, 터미널 안에서 파일을 편집할 수 있어야 한다. 대표 도구는 vi/vim과 nano다. Vim은 기능이 강력하지만 처음에는 어렵기 때문에, 강사는 vimtutor를 꼭 따라 해 보라고 한다.

sudo apt install vim
vimtutor
nano hello.txt
nano /etc/passwd

Ubuntu 20.04에서는 Vim이 이미 설치되어 있어 sudo apt install vim을 하지 않아도 vimtutor가 바로 실행될 수 있다. 설치 방법 자체는 뒤 패키지 강의에서 다시 배우므로, 여기서는 “편집기를 익혀야 한다”는 목적에 집중하면 된다.

vimtutor는 약 30분 정도면 끝까지 따라 할 수 있다. 한 번 했다고 바로 익숙해지지는 않으므로, 앞부분 5분 분량만이라도 며칠 반복해서 이동, 입력, 저장, 종료를 몸에 익히는 것이 좋다.

Nano는 Vim보다 단순하다. 화면 아래에 ^O, ^X 같은 단축키가 보이는데, 여기서 ^는 Control 키를 의미한다. 즉 ^O는 Ctrl+O로 저장, ^X는 Ctrl+X로 종료한다는 뜻이다. 강의 실습에서는 저장하지 않고 나가기 위해 Ctrl+X 후 N을 선택하는 흐름도 보여 준다.
